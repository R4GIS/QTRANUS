Results

This part of the program is used to produce maps with the results of a TRANUS model. In this manual we are using the model of Swindon as an example. This model and all related data may be downloaded from:

TRANUS site

To begin the process, a window is presented with boxes to fill:

Layers goup name

Give a name to put all the layers that you will produce into a group - Obligatory field

TRANUS workspace

A TRANUS project resides in a unique folder in your computer. As soon as you specify a valid TRANUS workspace, you will notice that the Scenario Tree appears with valid information in the left pane. This is because QTRANUS looks in the folder you specified for a file called TRANUS.CTL. This is the file where the scenario tree is specified, a fundamental structure in TRANUS modeling. You may click on the little + to expand the scenario tree

Zones shapefile

Specify the shape file where the polygons of the zones are contained. You may use the button provided to explore your pc. As soon as you put a valid shape file name, a map with the polygons should appear as a background.

The shapefile must contain one polygon for each zone in the model. Make sure that this is so. The first two fields in the related database must be zoneID, with a number assigned to each zone and zoneName, with a name. There must be a perfect match between the definition of zones in the shapefile and the definition of zones in the TRANUS model.

If no Zone shapefile is specified, the program will not be able to produce maps based on zonal information.

The rest of the boxes correspond to parts of the program that are still under development.

At the bottom there are three buttons called 'Zones', 'Network' and 'Matrix'. They are all inactive. As soon as a valid Zones Shapefile shapefile is specified, the Zones button becomes active. This is the only option available in this version. Press this button, and another window appears to generate QGIS layers with thematic zone-based results.

Once one or more map layers have been defined, you may save the QGIS project in the usual way, doing File-Save or File-Save As and giving it a name. This means that the next time you open the QGIS project, all your maps will be there exactly as you left them. You may then add more layers to the project using the QTRANUS plugin again and again.
